🚀 GigShield AI – Income Protection for Delivery Partners
📌 Overview
In India, delivery partners working with platforms like Zomato and Swiggy depend on daily earnings. However, their income is highly affected by external factors like heavy rain, extreme heat, or high pollution. During such situations, they are forced to stop working or receive fewer orders, which leads to a significant loss in their weekly income.
To solve this problem, we propose GigShield AI, an AI-powered parametric insurance platform that provides financial protection to delivery partners when such disruptions occur.

👤 Target Users (Persona)
We are focusing on food delivery partners.
Example Scenario
A delivery partner usually earns around ₹1000 per day.
But during:
->Heavy rainfall → fewer deliveries
->Extreme heat → reduced working hours
->High pollution → unsafe to work
This results in 20–30% income loss per week, and currently there is no system to support them.

💡 Our Solution
GigShield AI is designed to:
->Monitor external conditions like weather and pollution
->Detect when these conditions affect work
->Automatically provide compensation for income loss
The main idea is simple:
👉 If a disruption happens, the system will automatically trigger a payout.

⚙️ How the System Works
1. User Onboarding
The delivery partner enters basic details like:
->City
->Average weekly income
->Working hours
2. Risk Assessment (AI-Based)
The system analyzes:
->Weather patterns
->Location risks
Based on this, a risk score is generated.
3. Weekly Premium Calculation
A small weekly premium is calculated based on:
->Risk score
->Income level
4. Real-Time Monitoring
The system continuously checks:
->Rainfall
->Temperature
->Air Quality Index (AQI)
5. Trigger Event
->If any parameter crosses a defined threshold, it is considered a disruption.
6. Automatic Payout
->The system directly credits money to the user without requiring manual claims.

📊 Parametric Triggers
Some example conditions:
->Heavy Rain → Rainfall > 50mm
->Extreme Heat → Temperature > 42°C
->High Pollution → AQI > 300
->Curfew/Strike → Full work disruption
These triggers are fixed and transparent, so there is no delay in claim processing.

💰 Weekly Pricing Model
We are using a weekly subscription model since gig workers earn weekly.
Basic Idea:
Premium depends on:
->User income
->Risk level
Example:
If a user earns ₹6000/week and has medium risk,
their premium can be around ₹80–₹100 per week.

🤖 Use of AI/ML
We are using AI in two main areas:
1. Risk Prediction
The system predicts how risky a location is based on:
->Past weather data
->Environmental conditions
2. Fraud Detection
To avoid misuse:
->Location verification is checked
->Duplicate claims are blocked
->Unusual activity patterns are detected

🛠️ Tech Stack

->Database: MongoDB

Key Highlights:

Conclusion:
GigShield AI aims to provide a safety net for delivery partners by protecting their income from unpredictable external conditions. The platform ensures quick support, fairness, and ease of use, making it suitable for India’s growing gig economy.
